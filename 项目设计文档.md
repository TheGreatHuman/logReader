# 📄 项目设计文档：LogVision - 高性能日志可视化分析平台

## 1. 项目概述 (Project Overview)

### 1.1 项目背景
在工业控制、软件测试及系统运维场景中，会产生大量的日志数据。这些日志通常以压缩包形式归档，内含多个 CSV 格式的运行数据（时间序列）和文本格式的操作记录。现有的通用表格工具（如 Excel）无法流畅处理百万级数据点的缩放与交互，且缺乏将“运行数据”与“操作事件”关联分析的直观手段。

### 1.2 项目目标
开发一款基于 **.NET 8** 和 **WPF** 的 Windows 桌面应用 **LogVision**。该应用旨在提供：
*   **极速渲染**：轻松处理百万级数据点的折线图缩放与平移。
*   **关联分析**：将离散的“操作事件”精准标注在连续的“运行数据”时间轴上。
*   **高效管理**：对历史日志包进行导入、元数据编辑、存储与检索。
*   **双向联动**：实现图表与数据表的点击双向定位跳转。

## 3. 功能需求详述 (Functional Requirements)

### 3.1 日志包管理模块 (Log Package Management)
*   **导入流程**：
    1.  用户选择 `.zip` 文件。
    2.  系统自动解压至临时目录，扫描内部文件结构。
    3.  **文件选择对话框**：列出所有 `.csv` 文件，用户勾选需要可视化的文件（支持多选）。
    4.  **元数据编辑**：弹出窗口让用户输入/编辑日志名称、描述、测试人员、日期等标签。
    5.  **持久化存储**：
        *   将 ZIP 包内容（或选中的 CSV）移动至应用专用数据目录（如 `%APPDATA%/LogVision/Data/{GUID}/`）。
        *   解析 CSV 数据并写入 SQLite 数据库。
        *   记录元数据到 `Logs` 主表。
*   **列表管理**：
    *   左侧面板展示日志列表（卡片或列表视图）。
    *   支持操作：**重命名**、**编辑描述**、**删除**（级联删除数据库记录及本地文件）、**重新导入**（覆盖原有数据）。

### 3.2 数据存储与解析模块 (Data Engine)
*   **数据库设计策略**：
    *   采用 **单库多表** 或 **单库带 SessionID** 策略。推荐为每个导入的日志包创建一组表（或一个大表加 `LogPackageId` 索引），以便删除时直接 `DROP TABLE` 或 `DELETE WHERE Id = ...`，保证清理彻底且速度快。
    *   **表结构示例**：
        *   `RunData_{LogId}`: `Timestamp (REAL/INT64)`, `Col1`, `Col2`, ... `ColN`
        *   `OpEvents_{LogId}`: `Timestamp`, `OperationName`, `Description`, `Level` (Info/Warn/Error)
*   **解析逻辑**：
    *   识别 CSV 第一列为时间戳（毫秒级），自动转换为 `DateTime` 或保留为 `double` (OADate/Ticks) 以提升绘图性能。
    *   **自动分类**：根据文件名或列特征（如包含 "Op", "Cmd" 字样）自动区分“操作日志”与“运行日志”，或允许用户在导入时手动指定。

### 3.3 可视化交互模块 (Visualization Core)
*   **折线图绘制**：
    *   基于 ScottPlot 5，动态加载用户选择的列。
    *   支持多 Y 轴（不同量纲的数据在不同轴显示）。
    *   **性能优化**：利用 ScottPlot 内置的 `Resampler`，在缩放时自动计算可见范围内的最小/最大/平均值，确保百万数据点不卡顿。
*   **操作事件标注**：
    *   从 `OpEvents` 表读取数据。
    *   在折线图对应时间点绘制 **垂直标记线 (Vertical Line)** 或 **特殊 Marker (三角形/旗帜)**。
    *   鼠标悬停标记时，显示具体操作内容（Tooltip）。
    *   支持点击标记快速定位到该时间点。
*   **交互功能**：
    *   鼠标滚轮：缩放（Zoom）。
    *   鼠标拖拽：平移（Pan）。
    *   右键框选：局部放大。
    *   图例点击：隐藏/显示特定数据列。

### 3.4 联动与布局模块 (Linkage & Layout)
*   **三栏布局**：
    *   **左侧 (20%)**：日志包列表树/列表。
    *   **中间 (50%)**：ScottPlot 图表区域。
    *   **右侧 (30%)**：数据详情表格（DataGrid）。
    *   **调整**：使用 WPF `GridSplitter` 实现拖拽调整各区域宽度。
*   **双向联动 (Cross-Highlighting)**：
    *   **图 -> 表**：点击图表上的数据点（或标记），右侧 DataGrid 自动滚动并高亮显示对应的行。
        *   *实现*：监听 ScottPlot 的 `OnMouseClick` 事件，获取最近的数据点索引/时间，过滤 DataGrid 的 `ICollectionView` 并调用 `ScrollIntoView`。
    *   **表 -> 图**：点击 DataGrid 中的某一行，图表自动定位（Center On）到该时间点，并放大显示细节。
        *   *实现*：监听 DataGrid `SelectionChanged`，调用 ScottPlot 的 `AxisLimits` 设置中心点和缩放比例。

### 3.5 UI/UX 设计要求
*   **风格**：现代、简洁、深色模式优先（适合长时间查看数据）。
*   **反馈**：大数据加载时显示进度条（ProgressRing），避免界面假死。
*   **响应式**：窗口大小改变时，图表和表格自动重绘适应。

## 4. 系统设计 (System Design)

### 4.1 数据库 schema 设计 (SQLite)

```sql
-- 日志包元数据表
CREATE TABLE LogPackages (
    Id TEXT PRIMARY KEY, -- GUID
    Name TEXT NOT NULL,
    Description TEXT,
    OriginalFileName TEXT,
    ImportDate DATETIME DEFAULT CURRENT_TIMESTAMP,
    DataPath TEXT -- 本地存储路径
);

-- 动态生成的运行数据表 (示例：每个包一张大表，或用后缀区分)
-- 实际项目中可根据列数动态建表，或统一使用宽表
CREATE TABLE RunData_{LogId} (
    RowId INTEGER PRIMARY KEY AUTOINCREMENT,
    TimestampMs REAL NOT NULL, -- 毫秒时间戳，double 类型利于绘图
    -- 以下列动态添加
    Column_A REAL,
    Column_B REAL,
    ...
);

-- 操作事件表
CREATE TABLE OpEvents_{LogId} (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    TimestampMs REAL NOT NULL,
    OperationType TEXT, -- e.g., "START", "STOP", "ERROR"
    Message TEXT,
    RowReference INTEGER -- 可选：关联到 RunData 的行号
);

-- 索引优化
CREATE INDEX IDX_Timestamp_{LogId} ON RunData_{LogId}(TimestampMs);
CREATE INDEX IDX_OpTime_{LogId} ON OpEvents_{LogId}(TimestampMs);
```

### 4.2 核心类结构设计 (C#)

#### Model 层
*   `LogPackage`: 映射 `LogPackages` 表，包含元数据和文件路径。
*   `DataPoint`: 简单的 DTO，用于图表传输 `{ double X, double Y }`。
*   `OperationEvent`: 映射操作日志 `{ DateTime Time, string Message, MarkerType Type }`。

#### ViewModel 层 (MVVM)
*   `MainViewModel`: 统筹全局，管理当前选中的 `LogPackage`。
*   `LogListViewModel`: 处理左侧列表的加载、删除、重命名逻辑。
*   `ChartViewModel`:
    *   持有 `ScottPlot.Plot` 对象引用。
    *   方法：`LoadDataAsync(logId, selectedColumns)`, `AddAnnotation(event)`, `ZoomTo(time)`.
    *   属性：`SelectedTimeRange`, `VisibleColumns`.
*   `TableViewModel`:
    *   持有 `ObservableCollection<DataRow>` 或虚拟滚动数据源。
    *   方法：`SyncWithChart(double timestamp)`, `HighlightRow(int rowIndex)`.

#### View 层 (WPF)
*   `MainWindow.xaml`: 定义 Grid 布局，包含 `GridSplitter`。
*   `WpfPlot` 控件：嵌入 ScottPlot。
*   `DataGrid`: 启用虚拟化 (`VirtualizingStackPanel.IsVirtualizing="True"`) 以支持大数据量表格浏览。

### 4.3 关键业务流程：数据导入与解析

1.  **用户操作**: 拖入 ZIP 文件。
2.  **解压与扫描**:
    *   创建临时文件夹。
    *   使用 `ZipFile.ExtractToDirectory`。
    *   遍历文件，识别 CSV。
3.  **用户配置**: 弹窗展示 CSV 列表，用户勾选列和文件，输入日志名称。
4.  **后台处理 (Task.Run)**:
    *   在 SQLite 中创建新表结构。
    *   使用 `Sylvan.Data.Csv` 打开流。
    *   **分批读取** (每 10,000 行):
        *   解析时间戳和数值。
        *   如果是操作日志行，插入 `OpEvents` 表。
        *   如果是运行数据，批量 Insert 到 `RunData` 表 (使用 `SqliteTransaction` 提升速度)。
    *   更新 UI 进度条。
5.  **完成**:
    *   将文件移动到永久存储区。
    *   刷新左侧列表。
    *   自动加载新日志进入图表视图。

## 5. 关键技术难点与解决方案

### 5.1 百万级数据点的图表卡顿
*   **问题**: 直接渲染 100 万个点会导致 UI 线程阻塞。
*   **解决**:
    *   使用 **ScottPlot 5** 的内置信号绘图 (`Add.Scatter` 或 `Add.Signal`)，它会自动根据像素宽度进行**降采样 (Downsampling)**。例如屏幕宽 1000 像素，它只计算并绘制每个像素区间内的最大值和最小值，将渲染点数从 100 万降至 2000 左右，实现丝滑缩放。
    *   数据读取采用**分页加载**或**按需加载**，初始只加载概览数据，缩放到大级别时再查询数据库加载细节（可选高级优化）。

### 5.2 图表与表格的双向同步性能
*   **问题**: 点击图表某点，表格有 100 万行，如何快速定位？
*   **解决**:
    *   **不要遍历集合**。利用 SQLite 的索引，通过时间戳直接查询 `RowId`。
    *   WPF DataGrid 必须开启 **UI 虚拟化**。
    *   通过 `ICollectionView.MoveCurrentToPosition` 或直接设置 `DataGrid.SelectedIndex` 并调用 `ScrollIntoView`，这是 O(1) 或 O(logN) 的操作，不会卡顿。

### 5.3 操作日志的精准标注
*   **问题**: 操作时间点可能没有对应的运行数据点（时间未对齐）。
*   **解决**:
    *   ScottPlot 支持独立的 `Annotation` 或 `Marker` 图层，不依赖于折线数据源。
    *   直接从 `OpEvents` 表读取时间戳，在图表上绘制垂直线 (`Add.VerticalLine`) 和文本标签，无论该时间点是否有运行数据点，都能准确显示。

## 6. 开发计划建议 (Roadmap)

1.  **阶段一：原型验证 (PoC)**
    *   搭建 .NET 8 WPF 项目。
    *   集成 ScottPlot 5，测试 100 万随机数据的缩放和平移性能。
    *   集成 Sylvan 解析 CSV 并写入 SQLite。
2.  **阶段二：核心功能开发**
    *   实现 ZIP 导入、解压、元数据录入流程。
    *   完成数据库动态建表与数据入库。
    *   实现左侧列表管理和右侧 DataGrid 展示。
3.  **阶段三：交互与联动**
    *   实现操作日志的解析与图表标注。
    *   开发“图点表”和“表点图”的双向联动逻辑。
    *   实现三栏拖拽布局。
4.  **阶段四：UI 美化与优化**
    *   引入 ModernWpf 样式。
    *   增加加载动画、错误处理、日志导出功能。
    *   性能调优（内存泄漏检查、大数据压力测试）。
5.  **阶段五：发布**
    *   配置单文件发布 (`PublishSingleFile=true`)。
    *   安装包制作 (可选 MSIX 或 Inno Setup)。

## 7. 总结
本方案采用 **.NET 8 + WPF + ScottPlot 5** 的黄金组合，完美契合“大型 CSV 解析”与“高性能交互图表”的需求。通过 SQLite 进行轻量级数据持久化，利用 ScottPlot 的降采样技术解决渲染瓶颈，并通过 MVVM 模式保证代码的可维护性。该设计不仅能满足当前的日志可视化需求，还具备良好的扩展性，能够应对未来更复杂的数据分析场景。
