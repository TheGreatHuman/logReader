# Data - 数据访问层

本目录包含 SQLite 数据库的所有操作逻辑，负责日志元数据管理和运行数据的动态表操作。

## 文件说明

| 文件 | 说明 |
|------|------|
| `LogDatabase.cs` | 日志包元数据管理。数据库文件存储在 `%AppData%/LogVision/logvision.db`。管理 `LogPackages` 表的 CRUD 操作：创建表结构（`InitializeAsync`）、插入/查询/更新/删除日志包记录。每个日志包记录包含 ID、名称、描述、原始文件名、导入日期、数据路径和列名列表 |
| `DynamicDataAccess.cs` | 动态运行数据表操作。为每个导入的日志包动态创建独立的数据表（`RunData_{logId}` 和 `OpEvents_{logId}`）。支持：动态建表（根据 CSV 列名）、批量插入运行数据（每批 10000 行事务提交）、批量插入操作事件、按列加载图表数据、分页加载表格数据、按时间戳查找最近行、获取行数统计、删除表等操作。所有标识符都经过转义处理防止 SQL 注入 |

## 数据库结构

- **LogPackages 表**: 存储所有导入的日志包元信息（固定表）
- **RunData_{logId} 表**: 每个日志包的运行数据（动态创建，列由 CSV 决定）
- **OpEvents_{logId} 表**: 每个日志包的操作事件记录（动态创建）
